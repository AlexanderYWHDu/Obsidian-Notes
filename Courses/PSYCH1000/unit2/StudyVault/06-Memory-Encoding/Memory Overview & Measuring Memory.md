# Memory Overview & Measuring Memory (★★)

#memory #memory-encoding

## Overview Table

| Concept | Key Point |
|---------|-----------|
| Memory | Learning that persists over time |
| Encoding | Getting info into memory |
| Storage | Retaining info over time |
| Retrieval | Getting info back out |
| Recall | Retrieve without cues |
| Recognition | Identify from options |
| Relearning | How much faster you re-learn previously known info |

---

## What is Memory?

- **Memory** = learning that persists over time
- Three-stage process:
```
ENCODING → STORAGE → RETRIEVAL
(getting in)  (retaining)  (getting out)
```

---

## Measuring Memory: 3 Methods

### 1. Recall
- Retrieve information with **no cues** (or minimal cues)
- Example: short-answer exam, free recall of a word list
- Most demanding form

### 2. Recognition
- Identify previously learned information **from options**
- Example: multiple-choice exam
- Easier than recall (more cues available)

### 3. Relearning (Savings Method — Ebbinghaus)
- Re-learn something you already learned before
- **Savings score** = how much faster you relearn vs. original learning
- Shows memory exists even when recall/recognition fail
- Ebbinghaus's insight: **Forgetting is rapid early**, then levels off (forgetting curve)

```
EBBINGHAUS FORGETTING CURVE:

```

## Information-Processing Model

```
SENSORY MEMORY → WORKING/STM → LONG-TERM MEMORY
(milliseconds)  (20-30 sec)    (years)
(large capacity) (few chunks)  (virtually unlimited)
```

| Store | Duration | Capacity | Content |
|-------|----------|----------|---------|
| Sensory Memory | Milliseconds–seconds | Large | Raw sensory buffer |
| Working/STM | 20–30 s (rehearsed longer) | 7±2 items / few chunks | Active workspace |
| Long-Term Memory | Years | Virtually unlimited | Knowledge, skills, experiences |
